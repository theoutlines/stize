# design/gallery — слепок UI для Figma

Статичная HTML/CSS-галерея всех экранов и состояний приложения: одна страница на
состояние. Служит референсом текущего UI для работы над дизайном и **не является
продакшен-кодом**.

## Назначение

Галерея — слепок всех экранов и состояний UI для импорта в Figma. Владелец
забирает страницы плагином **html.to.design** (расширение Chrome): плагин
разбирает **DOM** страницы в слои и компоненты Figma. Flutter web рендерится в
canvas, поэтому само приложение плагину бесполезно — нужна DOM-реплика. **JS в
страницах запрещён**, внешних зависимостей нет, шрифты системные.

## Когда обновлять

Поддержание галереи — часть Definition of Done любой задачи, меняющей UI:

- **новое видимое состояние** → новая страница;
- **изменение существующего экрана** → правка его страницы;
- **удалённая фича** → страница удаляется из галереи и из `index.html`.

Задачи без визуальных изменений галерею не трогают.

## Правила страниц

- Фрейм **390×844** (iPhone), одно состояние на страницу.
- Имена файлов — **kebab-case**, говорящие (`stop-arrivals.html`,
  `map-line-focused.html`).
- **Семантичные имена классов** — html.to.design превращает их в имена слоёв
  Figma (`arrivals-sheet`, `arrival-tile`, `line-chip`, `vehicle-marker`…).
- Мок-данные реалистичные: линии и остановки **Белграда**, локализация как в
  приложении.
- Карта — статичная **подложка** (SVG/PNG), поверх — DOM-маркеры. Не растр для
  элементов, которые должны стать слоями.
- Состояния из **невлитых веток / спек** помечаются в `index.html` бейджем
  `branch` / `spec` (а фиче-флаг-состояния — `flag`). После merge ветки бейдж
  снимается.

## Деплой

Галерея живёт на Cloudflare Pages (проект `stigla`) на preview-алиасе
`preview-gallery`. Передеплой выполняет **сессия**, не владелец:

```sh
npx wrangler pages deploy design/gallery --project-name=stigla --branch=preview-gallery
```

- **URL:** https://preview-gallery.stigla.pages.dev
- Превью под тем же паролем (Basic Auth, `_worker.js`), что и остальные
  `*.pages.dev`-превью; прод-домен не затронут. Доступы — см. `docs/staging.md`.

## Отчётность

В отчёте задачи, менявшей UI, — отдельная строка про галерею:
«Галерея: обновлены страницы …» либо «Галерея: визуальных изменений нет».
