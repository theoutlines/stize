# Отчёт: HTML-галерея экранов и состояний для импорта в Figma

**Дата:** 2026-07-12
**Ветка:** `feature/design-gallery`
**Задача:** `docs/prompts/prompt_design_gallery.md`

## Что сделано

В `design/gallery/` собрана статичная HTML/CSS-реплика всех экранов и состояний
приложения — по одной странице на состояние, мобильный фрейм **390×844**,
реалистичные мок-данные Белграда. Это референс текущего UI для редизайна, **не
продакшен-код**. Приложение (`app/`) и бэкенд (`backend/`) не тронуты.

Галерея делается ради плагина **html.to.design** (Chrome): он разбирает **DOM**
страницы в слои Figma. Flutter web рендерится в canvas, поэтому само приложение
плагину бесполезно — нужна DOM-реплика. Всё, что должно стать слоями (маркеры,
пилюли ТС, чипы, шторки, бейджи), — это DOM-элементы с семантичными именами
классов (`stop-pin`, `vehicle-marker`, `arrivals-sheet`, `line-chip`…), которые
плагин превращает в имена слоёв. JS и внешних зависимостей в страницах нет,
шрифты системные, карта — статичная нейтральная SVG-подложка, поверх — DOM.

### Решения по развилкам (согласованы владельцем)

- **Язык UI:** английский (EN) — как EN-локализация приложения; географические
  названия остаются сербскими.
- **Тема:** только тёмная (токены — приближение Material 3 dark из seed-цвета
  `#2E7D6B`; цвета типов транспорта взяты дословно из `lib/core/map_support.dart`).
- **Подложка карты:** нейтральная стилизованная заглушка (`assets/map-bg-dark.svg`),
  без реальных скриншотов — не требует запуска приложения и ключа MapTiler.
- **Флот (B2/B3/B4):** свёрстан по спеке, помечен бейджем `spec` (см. ниже).

### Находка по шагу 1б (ветки)

Проверил живые ветки. **`feature/fleet-id` уже влита в `main`** (является её
предком) и содержит только заготовки аналитики — **UI флота в коде нет нигде**
(в `arrival_tile.dart` бейджей нет). Поэтому компакт-бейджи (B2), карточка модели
(B3) и сортировка «по комфорту» (B4) сверстаны целиком по
`docs/FLEET_INTEGRATION.md` и помечены `spec`. Ветки `transport-analytics` и
`coverage-on-main-map` тоже влиты — их состояния уже присутствуют в main (за
фиче-флагами). Единственная ветка с реально новым UI —
**`feature/nearby-list`** (список «Рядом» за флагом `nearby_list`), её состояния
взяты из кода ветки и помечены `branch`.

## Структура

- `index.html` — оглавление со ссылками на все состояния (плагин импортирует
  постранично); карточки с живыми превью, сгруппированы, с легендой бейджей.
- `styles.css` — общие дизайн-токены тёмной темы и все семантические классы.
- `assets/map-bg-dark.svg` — нейтральная подложка карты.
- `_worker.js` — тот же парольный гейт превью, что и у остальных `*.pages.dev`.
- по файлу на состояние (kebab-case).

Страницы сгенерированы build-time скриптом; **в самих страницах JS отсутствует**.

карточки оглавления, чтобы было видно: это не текущий прод.

## Деплой

`design/gallery/` задеплоена на Cloudflare Pages (проект `stigla`) на
preview-алиас ветки `preview-gallery`:

- **URL галереи:** https://preview-gallery.stigla.pages.dev
- Превью под тем же **паролем**, что и остальные `*.pages.dev` (Basic Auth,
  тот же `_worker.js`; прод-домен не затронут).

Пере-деплой (силами сессии, владельцу не нужен терминал):
`npx wrangler pages deploy design/gallery --project-name=stigla --branch=preview-gallery`.

## Как проверить (для владельца, без терминала)

1. **Открой галерею в Chrome:** https://preview-gallery.stigla.pages.dev
   - Появится запрос логина/пароля (Basic Auth). **Логин** `stigla`, **пароль** —
     из командного менеджера паролей (там же, где доступы к остальным
     `*.pages.dev`-превью; подробности — `docs/staging.md`, раздел «Per-branch
     previews»). Браузер обычно подставит их сам.
2. **Оглавление** (`index.html`) показывает все состояния карточками с живым
   превью, сгруппированные (Карта / Остановка / Навигация / За флагом / Ветка /
   Флот). Клик по карточке открывает конкретное состояние на отдельной странице.
   Цветные бейджи: синий `flag` — есть в проде, но выключено; фиолетовый
   `branch` — не в main; оранжевый `spec` — только дизайн, ещё не реализовано.
3. **Импорт в Figma (постранично):** установи расширение Chrome
   **html.to.design**, открой нужную страницу состояния (например
   `.../stop-arrivals.html`), запусти плагин на этой вкладке — он разберёт DOM в
   слои и компоненты. Импортируй по одной странице (оглавление для этого не
   импортируй — только конкретные состояния). Имена слоёв в Figma — это имена
   CSS-классов (`arrivals-sheet`, `arrival-tile`, `line-chip`, `vehicle-pill`…).
4. **Что должно быть видно на страницах (выборка):**
   - `map-default` — тёмная карта, пины остановок (синий автобус / красный трамвай
     / оранжевый троллейбус / серый микс / жёлтая звезда-избранное), пилюли
     движущихся ТС с «клювом» направления и подсветкой, кластер «6», точка «моя
     позиция», внизу карусель избранного + строка поиска.
   - `map-line-focused` — подсвеченный красный маршрут трамвая 12, только его
     остановки, нижняя панель линии «Omladinski stadion → Banovo brdo».
   - `stop-arrivals` — шторка «Slavija»: алерт, метка свежести «Updated just now»,
     чипы-фильтр линий (активные + приглушённая неактивная «2»), тайлы прибытий с
     бейджами изменения ETA (зелёный ↓, оранжевый ↑).
   - `analytics-line` — хитмап час×день, спарклайн (2 линии сравнения), дот-плот,
     плашки-метрики (бейдж `flag`, текст на русском — как в приложении-черновике).
   - `coverage` — тепловая карта плотности (Strava-стиль), чипы фильтра типов,
     легенда плотности (бейдж `flag`).
   - `nearby-list` — шторка «Рядом»: карточки линия→направление с ETA (бейдж
     `branch`).
   - `fleet-model-card` — карточка «Ката» (Tatra KT4YU): nickname крупно, модель
     мелко, точки комфорта, атрибуты (нет кондиционера, ~44 года, высокий пол),
     схема салона-заглушка (бейдж `spec`).
   - `stop-sort-comfort` — та же остановка с переключателем сортировки «By
     comfort»: «Ката через 3 мин или Bozankaya через 9» (бейдж `spec`).
5. Если какие-то состояния лишние/нужно добавить — скажи, поправлю и передеплою.

## Ограничения соблюдены

Секретов, ключей и деталей upstream-источника в галерее нет. Merge в `main` не
делался — только по явной команде.
