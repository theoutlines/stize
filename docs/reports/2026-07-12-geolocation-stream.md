# Отчёт: живая геолокация пользователя (стрим вместо one-shot)

Ветка: `fix/geolocation-stream`. Задача: `docs/prompts/prompt_fix_geolocation_stream.md`.

## Диагноз

Гипотеза («геолокация запрашивается one-shot внутри общего refresh-цикла карты»)
**подтвердилась по сути, но с уточнением по механике**:

- Позиция пользователя хранилась в поле `_myPosition` (`home_map_screen.dart`) и
  обновлялась **только** через `getCurrentPosition()` — one-shot фикс.
- Вызывался этот путь ровно в двух местах: `initState()` (один раз при входе) и
  на кнопке recenter (`_recenterOnMe`). **К 30-секундному таймеру опроса
  транспорта (`_vehiclesTimer`) он привязан не был.**
- Итог: во время поездки маркер вообще не двигался — он «замирал» на первом
  фиксе и оживал только по нажатию кнопки «моё местоположение». Совпадение с
  30-секундным ритмом — визуальная иллюзия (единственный источник обновления —
  ручное нажатие), но корень тот же, что в гипотезе: **нет подписки на
  позиционный стрим**. Поэтому [STOP & ASK] не потребовался — направление
  правки совпало с постановкой.

Прочие места чтения позиции: `lastKnownIfGranted()` (мгновенное центрирование на
старте) и `getCurrentPosition()` — оба в `LocationService`. Второго источника
позиции в коде нет.

## Что сделано

### 1. Непрерывный стрим позиции
- `LocationService.positionStream()` — обёртка над
  `Geolocator.getPositionStream(...)`. Платформенные исключения нормализованы в
  `LocationUnavailable`, чтобы виджет не тянул типы geolocator.
- Настройки стрима вынесены рядом с существующими one-shot-настройками
  (`location_settings_io.dart` / `location_settings_web.dart`):
  `buildStreamLocationSettings()` — **accuracy: high, distanceFilter: 8 м**, без
  `timeLimit` (стрим не должен «сдаваться»); на web `maximumAge: 0` (нужны
  свежие фиксы). Обновления приходят **по перемещению, а не по таймеру** — это
  локальный GPS, правило «поллинг ≥30с» на него не распространяется (оно только
  про вызовы нашего API).

### 2. Маркер и сглаживание
- Маркер «моё местоположение» переведён на стрим. Между фиксами он **плавно
  подъезжает** к свежему фиксу коротким easing (`_meEaseDuration = 700 мс`,
  линейно — читается как равномерное движение).
- Сглаживание реализовано отдельно, **без переиспользования
  `vehicle_track_animator`** (у него семантика «не обгонять последний фикс» для
  редких обновлений; здесь фиксы частые). Чистая функция `lerpLatLng` вынесена в
  `core/user_location_tracker.dart` и покрыта тестами.
- Маркер завёрнут в `AnimatedBuilder(_meAnim)`: тикер работает **только во время
  easing**, между фиксами простаивает — лишних перерисовок нет.

### 3. Жизненный цикл подписки
- Стрим живёт по правилу `active = видимая вкладка карты && приложение на
  переднем плане && разрешение не отозвано`. Reconcile-логика
  (`_reconcileLocationStream`) сводит фактическое состояние подписки к этому
  правилу.
- **Фон / скрытая вкладка**: `WidgetsBindingObserver.didChangeAppLifecycleState`
  ловит `resumed`/прочее (на web это же покрывает `visibilitychange`) — подписка
  ставится на паузу и возобновляется при возврате.
- **Экран Ideas**: `RootScreen` держит обе страницы в `IndexedStack`, поэтому в
  `HomeMapScreen` добавлен флаг `active` (`_index == 0`); при уходе на Ideas
  стрим останавливается — батарея не жжётся.
- **Молчащий `watchPosition` (web / iOS Safari)**: чистая функция
  `shouldResubscribe(...)` решает, не «завис» ли стрим — если экран активен и
  фиксов не было дольше 15 с (порог считается от позднейшего из «последний фикс»
  и «момент подписки»), подписка **пересоздаётся**. Покрыто тестами.
- **Отзыв разрешения на лету**: ошибка стрима с permission/serviceDisabled →
  маркер скрывается (`_meTo = null`), подписка останавливается, флаг
  `_locationDenied` не даёт спамить повторным запросом. Заново разрешение
  запрашивается **только** кнопкой recenter (реальный жест пользователя).

### 4. Смежное не сломано
- Кнопка recenter (`_recenterOnMe`) переведена на последний фикс из стрима
  (`_meTo`) для мгновенного отклика; затем делает один `getCurrentPosition`
  (он же вызывает системный промпт при первом использовании) и
  до-запускает/переактивирует стрим. Второго источника позиции не появилось.
- Начальное центрирование (`_initLocation`): при уже выданном разрешении —
  мгновенно по `lastKnownIfGranted`, затем стрим; первый фикс центрирует камеру
  один раз (`_pendingRecenter`), последующие — только двигают маркер, не воюя с
  панорамированием пользователя.

## Проверка руками (владелец, iPhone)
1. **Плавность в поездке.** Открыть карту, разрешить геолокацию, сесть в
   транспорт. Ожидание: точка едет **непрерывно и плавно**, подъезжая к каждому
   фиксу, без 30-секундных «телепортов». Движение маркера не связано с
   обновлением транспорта.
2. **Свернуть/развернуть Safari.** Свернуть вкладку/приложение на 20–30 с,
   вернуться. Ожидание: маркер **оживает** в течение ~несколько секунд (стрим
   пересоздаётся по watchdog), а не остаётся замороженным.
3. **Уход на Ideas.** Переключиться на Ideas и обратно — карта продолжает
   отслеживать позицию; на Ideas стрим не работает (батарея).
4. **Отзыв разрешения.** В настройках iOS отозвать доступ к геолокации при
   открытой карте. Ожидание: маркер тихо исчезает, без ошибок и без повторных
   всплывающих запросов; кнопка «моё местоположение» снова спрашивает доступ.
