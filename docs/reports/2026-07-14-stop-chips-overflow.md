# Чипы-фильтр линий на экране остановки: устойчивость к большому числу линий

**Дата:** 2026-07-14
**Ветка:** `fix/stop-chips-overflow`
**Тип:** мелкая UI-правка (только чипы фильтра; данные/карта/список прибытий не тронуты)

## Проблема

На экране остановки под плашкой свежести показан ряд чипов-фильтров по линиям
(«Все линии» + чип на каждую линию остановки; неактивные — приглушённые). Ряд
был свёрстан через `Wrap`: при малом числе линий это одна строка, но у крупных
пригородных остановок после вливания пригорода линий стало много — у **Baćevac
~22**. `Wrap` переносил их на 4–5 строк, и этот высокий блок сдвигал список
прибытий далеко вниз (на маленьких экранах — почти за пределы видимой части
шторки). Фильтр из помощника превращался в стену.

## Что сделано

Единственная правка в коде — блок чипов в
[stop_screen.dart](app/lib/presentation/screens/stop_screen.dart:230):
`Wrap(spacing: 8)` заменён на горизонтально скроллящийся
`SingleChildScrollView(scrollDirection: Axis.horizontal)` с `Row(spacing: 8)`.

- Много линий → один аккуратный ряд, который **скроллится вбок**; список
  прибытий остаётся сразу под ним и не уезжает.
- Мало линий → тот же вид, что и раньше: ряд просто не скроллится.
- Логика фильтрации не менялась: `ChoiceChip` с `selected`/`onSelected`,
  «Все линии», приглушённые (`Opacity 0.4`, `onSelected: null`) неактивные
  линии — всё как было.

Паттерн — тот же, что у карусели избранного (F7): горизонтальный скролл с
краевыми отступами, чтобы крайние чипы не обрезались вплотную.

### Галерея

- Обновлён CSS `.line-filter` в
  [styles.css](design/gallery/styles.css:475): `flex-wrap: wrap` →
  `nowrap` + `overflow-x: auto`, чипы не сжимаются (`flex: 0 0 auto`,
  `white-space: nowrap`) — зеркалит поведение приложения.
- Добавлена новая страница состояния
  [stop-many-lines.html](design/gallery/stop-many-lines.html) — остановка
  Baćevac с ~22 линиями: фильтр одной скроллящейся строкой, обрезанной по
  правому краю. Зарегистрирована карточкой «Many lines» в
  [index.html](design/gallery/index.html) (группа «Stop arrivals»).
- Существующая `stop-arrivals.html` не правилась — новый CSS оставляет её
  ряд из 7 чипов одной строкой.

  месте.

## Как проверить (без терминала)

Галерея-слепок UI задеплоена на превью:

1. Открыть **https://preview-gallery.stigla.pages.dev** (пароль превью — см.
   `docs/staging.md`).
2. В группе **«Stop arrivals»** появилась карточка **«Many lines»** — открыть её.
3. Ожидаемо: остановка **Baćevac**, под «Updated just now» — ряд чипов линий
   **одной строкой**, обрезанный по правому краю (это ~22 линии; на реальном
   устройстве ряд листается пальцем вбок). Список прибытий — сразу под чипами,
   не уехал вниз.
4. Для сравнения соседняя карточка **«Arrivals»** (остановка Slavija, мало
   линий) — тот же ряд, тоже одной строкой, вид как раньше.

На самом устройстве это увидит тот, кто откроет крупную пригородную остановку
(например, Baćevac): чипы линий листаются вбок, а не громоздятся в высокий блок.
