# Символьный GPU-слой ТС — непрерывность движения, фикс хорды, мерж в прод

**Дата:** 2026-07-14
**Ветка:** `feature/symbol-layer` → слита в `main`
**Статус:** в проде

Продолжение задачи символьного слоя (разработка — в
`docs/reports/2026-07-13-symbol-layer.md`). В этой сессии закрыты две
регрессии движения, замеченные на телефоне, и ветка выведена в прод.

## Что сделано

### 1. Непрерывное движение (как в референс-приложениях)
**Симптом:** между 30с-опросами машина ехала ~15с, потом досиживала неподвижно
до следующей пачки данных — весь экран «замирал и оживал» разом.

**Причина:** маркер вёл лишь короткий «мостик» (15с) вперёд от фикса, дальше
держался. Мостик был ошибочно взят как защита от «пролёта».

**Фикс:** защита от пролёта — это **гейт устаревания**, а не короткий мостик.
Пока фикс свежий (возраст ≤ 45с), предсказываем **всё** прошедшее время →
машина едет непрерывно весь интервал опроса. Как только борт протухает
(возраст > 45с, данные не обновляются) — цель схлопывается к фиксу, маркер
**держится** (только-вперёд, не откатывается и не дёргается), улететь не может.
Гейт (45с) стоит выше периода опроса (30с): здоровые данные никогда не «сидят»,
замёрзший борт пришпилен в пределах одной ширины гейта. Дистанционный пояс
(`_maxAheadMeters` 500→900м) — страховка от неправдоподобно быстрого плана.

### 2. Ход по геометрии улицы, а не хордой сквозь дома
**Симптом:** машина после появления ехала по прямой между точками плана, срезая
изгибы улиц («сквозь дома») ~30с, потом вставала на дорогу.

**Причина:** до загрузки дорожного shape маркер использовал фолбэк-геометрию
(точки плана = хорды). `_ensureShapesFor` клал shape в кэш, но **не
переприменял** его к живым timed-трекам — те висели на хорде до следующего
30с-опроса.

**Фикс:** при загрузке shape срабатывает дебаунснутый (~120мс) ре-синк —
timed-плеер апгрейдит геометрию хорда→дорога за ~латентность фетча,
ре-анкор без прыжка (`updatePlan`). Окно хорды схлопнуто с ≤30с до ~1с.

### 3. Появление без рывка
Расширение окна предсказания грозило «догоняющим рывком» вперёд у только что
появившейся машины (заметно при панораме). Маркер теперь появляется в
**предсказанной-сейчас** точке (`plan[возраст]`, тот же гейт) — там, где ТС
реально сейчас; протухший борт по-прежнему появляется на фиксе и стоит.

**Побочно:** ушло дёрганье меток на статичной карте — старый 15с-мостик создавал
флип «едет→стоит» каждый интервал, от которого spiderfy мигал веером; при
непрерывном движении окна нет. Фоновый режим (термо-пауза `ios-web-thermal`) не
тронут — непрерывность работает только на переднем тикере.

## Мерж и деплой в прод (порядок выполнения)
1. `main` актуализирован (`git pull --ff-only` → up to date).
2. `feature/symbol-layer` слит в `main` (`--no-ff`, merge-коммит `b22229d`) —
   **конфликтов нет** (main ушёл вперёд лишь на docs-промпт).
3. Тесты на слитом main: **клиент 155 / backend 83 — зелёные**, `tsc --noEmit` чист.
4. Backend на прод: `npm run deploy` → version `c1dde62e`.
5. Флаги на прод (прямая запись в прод-KV, `--remote`):
   `flag:symbol_layer=1`, `flag:timed_trajectory=1`. Прод `/api/v1/config`
   подтверждает оба `true`.
6. Web на прод: `flutter build web --release --dart-define-from-file=dart_defines.json`
   (без staging-дефайнов) → `wrangler pages deploy build/web --branch=main`.
   Бандл вшил прод-бэкенд (`stigla-api.theoutlines.xyz`).
7. Верификация по sha (не браузером — зона переписывает cache-заголовки):
   `stigla.theoutlines.xyz/main.dart.js` sha256 == локальный `build/web/main.dart.js`
   (`606dd4e5…`) — **MATCH**.
8. Данные прода: `/vehicles/nearby` отдаёт 34 ТС с `route_id`+`as_of`+`trajectory`
   (14 точек), все `source:live` — расписание на карту не течёт (Фаза 2 не в
   ветке).

## Откат (если что-то не так)
Мгновенный, без редеплоя: выключить флаг —
```
npx wrangler kv key put --remote --namespace-id=9908db7cfc444d6692a66b2577b91288 "flag:symbol_layer" "0"
```
Клиент на следующем 30с-опросе (или перезагрузке) вернётся на старый виджетный
путь (он остался фолбэком за флагом). `timed_trajectory` можно гасить отдельно.

## Как проверить (владельцу, без терминала)
Открой **https://stigla.theoutlines.xyz** (прод; бейджа STAGING быть **не
должно** — это прод-сборка). На карте, зумни на улицу с трамваем/автобусом:
1. **ТС рисуются символьным слоем** (пилюли на детальном зуме / точки на общем),
   без лагов при большом числе машин.
2. **Движение непрерывное** весь интервал — нет «замер на 15с». Весь экран не
   замирает-оживает разом.
3. **Едут по изгибам улиц**, не срезают по прямой сквозь дома — в т.ч. сразу
   после появления и при панораме (без рывка-догоняния).
4. Если данные подвиснут — машина встаёт, не улетает.

Если что-то из этого не так — сообщи, откатим флагом за секунду (см. выше) и
разберёмся, прод при этом останется рабочим.
