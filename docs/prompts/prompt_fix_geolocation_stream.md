# Задача: живая геолокация пользователя (стрим вместо one-shot)

Ветка: `fix/geolocation-stream`. Код и комментарии — EN, отчёт — RU в
`docs/reports/YYYY-MM-DD-geolocation-stream.md`.

## Контекст / симптом

Маркер позиции пользователя на карте обновляется рывками, визуально синхронно
с 30-секундным циклом опроса транспорта: пользователь едет, а его точка стоит
и «телепортируется» раз в цикл. Ожидание: позиция обновляется непрерывно и
независимо от поллинга бэкенда.

Гипотеза: геолокация запрашивается one-shot (`getCurrentPosition` или аналог)
внутри общего refresh-цикла карты, вместо подписки на позиционный стрим.
**Сначала подтвердить или опровергнуть гипотезу чтением кода** (`home_map_screen.dart`
и всё, что трогает location). Если реальная причина другая — [STOP & ASK]
с найденным диагнозом до начала правок.

## Что сделать

1. **Диагноз.** Найти все места, где читается позиция пользователя. Зафиксировать
   в отчёте: какой механизм используется сейчас, к какому таймеру привязан.

2. **Стрим позиции.** Перевести маркер пользователя на непрерывную подписку
   (position stream текущего location-пакета проекта):
   - accuracy: high; distanceFilter ≈ 5–10 м (не по таймеру);
   - подписка живёт отдельно от цикла опроса транспорта. Правило «поллинг ≥30с»
     относится **только** к вызовам нашего API и на локальный GPS не распространяется;
   - разумное сглаживание маркера допустимо (короткое easing к свежему фиксу),
     но не тянуть сюда `vehicle_track_animator` без необходимости — у него
     семантика «не обгонять последний фикс» для редких обновлений, здесь фиксы частые.

3. **Жизненный цикл подписки.**
   - Пауза стрима, когда карта не видна / приложение в фоне; возобновление при
     возврате. Не жечь батарею на экране Ideas.
   - Web / iOS Safari: `watchPosition` может молча замирать после потери
     видимости вкладки. На событие возврата видимости (`visibilitychange` /
     resumed lifecycle) — пересоздавать подписку, если фиксов не было дольше
     порога (например 15с при активном экране).
   - Отзыв разрешения на лету → маркер скрывается без ошибок, повторный запрос
     разрешения не спамится.

4. **Не сломать смежное.** Если центрирование «по моей позиции» или что-то ещё
   переиспользует старый one-shot путь — перевести на последний фикс из стрима,
   не плодить второй источник.

## Тесты

- Юнит: логика «стейл-фикс → пересоздать подписку» и easing (если добавлен) —
  вынести в чистые функции по образцу `vehicle_track_animator`.
- Widget-тесты экрана карты — с `kMapRenderingEnabled = false` (см. CLAUDE.md).
- `flutter analyze lib test` чистый.

## Проверка руками (владелец, iPhone)

Описать в отчёте сценарий проверки: поездка на транспорте с открытым экраном,
ожидание — маркер движется плавно, без 30-секундных прыжков; сворачивание/
разворачивание Safari — маркер оживает.

## Развилки

- Реальная причина не совпала с гипотезой → [STOP & ASK].
- Текущий location-пакет не даёт нужного API на web → [STOP & ASK] с вариантами
  (условная реализация через `dart:js_interop` / смена пакета), не выбирать молча.
