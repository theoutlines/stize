# Задача: карта покрытия транспортом (Coverage Map, V0)

Ветка: `feature/coverage-map`, worktree `../stigla-coverage-map`.
Спека: `docs/COVERAGE_MAP.md` — прочитать целиком до начала работы.
Контекст проекта: `CLAUDE.md` (архитектура, gotchas), `docs/BACKLOG.md`.

## Что сделать

Новая вкладка-инфографика: светящиеся коридоры маршрутов поверх обычной
подложки карты. V0 по спеке: предвычисленный GeoJSON из GTFS shapes +
line layer с data-driven шириной/яркостью + фильтр по типу ТС + фиче-флаг.

## Порядок работы

### 1. Скрипт сборки данных покрытия (`backend/`)

- Расширить пайплайн `gtfs:build` (или добавить соседний скрипт
  `scripts/build-coverage.mjs`, вызываемый из него): shapes всех маршрутов →
  сегменты → схлопывание совпадающих сегментов с допуском → GeoJSON
  с properties `routes_count` (int), `types` (array: tram/trolleybus/bus),
  и заделом под будущие веса (см. спеку).
- Результат кладётся в `public/gtfs/` рядом с остальной статикой.
- Если файл > ~1–2 МБ gzip — simplify геометрии на этапе сборки.
- [STOP & ASK] если метод схлопывания сегментов даёт визуально грязный
  результат (двоящиеся линии, разрывы) и нужен выбор между вариантами
  алгоритма — показать варианты с trade-offs, не выбирать молча.

### 2. Раздача

- Помнить gotcha из CLAUDE.md: новый путь под `/gtfs/` раздаётся статикой
  и обходит CORS-middleware. Если клиенту нужен fetch с другого origin —
  явный Hono-роут. Проверить fetch из web-сборки.

### 3. Экран покрытия (`app/`)

- Третий экран в `IndexedStack`, пункт в drawer, роут в `router.dart`.
- Подложка — существующий `core/map_style.dart` (theme-synced), поверх —
  line layer из GeoJSON: ширина и цвет — expressions от `routes_count`.
  Цветовые шкалы: тёплая (оранжевый → белый) на тёмной теме, читаемый
  аналог на светлой — на усмотрение исполнителя по критерию из спеки.
- Чипы фильтра по типу ТС (мультивыбор), фильтрация через expression
  на слое, без пересборки источника.
- Легенда шкалы веса.
- Учитывать `kMapRenderingEnabled` для тестов (см. CLAUDE.md).

### 4. Фиче-флаг

- `coverage_map_show` (KV, env-aware, по образцу `analytics_show`):
  OFF на проде, ON на staging. Флаг прячет пункт drawer и роут.

### 5. Тесты и приёмка

- Юнит-тесты на скрипт сборки: схлопывание сегментов, счёт маршрутов,
  типы ТС; edge cases (маршрут с одним shape, полностью совпадающие маршруты).
- Widget-тест экрана с `kMapRenderingEnabled=false`.
- `flutter analyze lib test`, `npx tsc --noEmit`, оба тест-раннера зелёные.

## Definition of Done

1. Всё по DoD спеки `docs/COVERAGE_MAP.md`.
2. Код закоммичен в `feature/coverage-map`, тесты проходят.
3. Отчёт RU в `docs/reports/2026-07-XX-coverage-map.md`: что сделано,
   размер итогового GeoJSON, скриншот-инструкция для визуальной проверки
   на iPhone, открытые вопросы.
4. Merge в main — только по явной команде владельца.

## Ограничения

- Публичный репозиторий: никаких секретов и деталей upstream-источника.
- Бэкенд-рантайм не трогаем (только скрипт сборки и, при необходимости,
  CORS-роут). Никакого поллинга — слой статичный.
- Не выходить за V0: частоты, час дня, glow-эффекты — вне задачи.
