# Задача: сломанные остановки (нет кликабельной) + иконка трамвайной остановки

Ветка: `fix/stop-data` · Worktree: `../stigla-stop-data`
Отчёт: `docs/reports/2026-07-XX-stop-data.md` (RU)

Два бага данных остановок (не рендер/не движение). Оба существуют с ранних
версий — то есть в данных/сборке, а не в недавних изменениях. Оба бьют по
доверию на запуске: пустая остановка «напротив дома» и трамвайная остановка с
автобусной иконкой — заметны сразу.

## БАГ 1 — остановка есть на подложке, но кликабельной нет
Симптом: на карте видна иконка остановки (от MapTiler/OSM-подложки), но **тапнуть
и открыть прибытия нельзя** — своей (из нашего GTFS) остановки там нет.

**Конкретный тест-кейс (владелец):** остановки **«Батутова»** — обе на **бульваре
Краља Александра** (на пересечении с ул. Батутова, отсюда название), обе
обслуживают **автобус 79**, оба направления. На подложке видны две иконки
«Батутова», но кликабельной остановки из нашего GTFS там нет. Место: бул. Краља
Александра × Батутова (Zvezdara). По этим двум остановкам и вести диагностику 1a
vs 1b.

Механика: два разных слоя остановок —
- иконка подложки (OSM, часть картинки карты, не наши данные);
- наша кликабельная остановка из `public/gtfs/stops.json` (по ней тап открывает
  прибытия).
Если иконка есть, а клика нет → остановки **нет в нашем GTFS-бандле**.

### Диагностика (до фикла, [STOP & ASK])
Два возможных корня — разные фиксы:
- **1a — остановки НЕТ в исходном GTFS-фиде** (город её не включил). Тогда наш
  бандл честно её не содержит; фикс — вне нашего контроля (или доб. вручную/из
  OSM — обсудить).
- **1b — остановка ЕСТЬ в фиде, но отфильтровалась при сборке** (`build-gtfs.mjs`
  выкинул — напр. остановка без прибытий/линий, без родительской станции, дубль).
  Тогда фикс — поправить фильтр сборки.
Взять КОНКРЕТНЫЕ остановки владельца (координаты/название/улица) → проверить:
есть ли они в исходном GTFS `stops.txt`? если да — почему не дошли до
`stops.json` (какой шаг сборки отсёк)? Локализовать 1a vs 1b. **[STOP & ASK]** с
вердиктом и предложением фикса до реализации.

## БАГ 2 — трамвайной остановке присвоена не трамвайная иконка
Симптом: трамвайная остановка (на путях) получает **смешанную** иконку
(автобус+троллейбус) вместо трамвайной, потому что к ней подходят и ночные
автобусы. Правило владельца категорично: **трамвайная остановка = трамвайная
иконка, всегда.** Трамвай доминирует; наличие ночных автобусов не отменяет
трамвайности.

### Фикс
Найти классификацию типа остановки (по набору обслуживающих линий → иконка;
вероятно `map_support.dart` / где строится тип пина остановки). Ввести **приоритет
типов**: если остановку обслуживает трамвай — иконка трамвайная, независимо от
наличия автобусных/троллейбусных линий. Порядок доминирования обсудить, но
трамвай точно выше автобуса. Проверить, что чисто-автобусные/троллейбусные
остановки не затронуты.

## Границы
- Только данные остановок (покрытие + классификация типа/иконки). НЕ трогать:
  рендер движущихся ТС (символьный слой), расписание, живой источник.
- Не пересекаться с `feature/symbol-layer` (движение) и `feature/gtfs-freshness`
  (расписание) — это про статичные остановки, другие файлы.
- Если БАГ 1 = 1b (фильтр сборки) — правка `build-gtfs.mjs`, пересборка бандла,
  [STOP & ASK] перед прод-пересборкой (как с пригородом).
- Код EN, отчёт RU, без секретов.

## Definition of Done
1. Отчёт RU: по БАГУ 1 — вердикт 1a/1b на конкретных остановках владельца, где
   теряется; по БАГУ 2 — правило приоритета типов. Таблицы «остановка → есть в
   фиде? → в бандле? → почему» и «тип линий → иконка».
2. Фиксы: БАГ 1 (если 1b — фильтр сборки), БАГ 2 (приоритет трамвая). Тесты:
   классификация иконки (трамвай+автобус→трамвай; чистый автобус→автобус),
   и если 1b — тест, что остановка-кейс теперь в бандле.
3. Меняется вид карты (иконки остановок) → обновить `design/gallery/` (markers-
   legend / состояния с остановками) + передеплой preview-gallery.
4. «Как проверить» без терминала: staging-превью сессия деплоит сама; владельцу —
   открыть конкретные остановки напротив дома (кликабельны?) и трамвайную
   (трамвайная иконка?).
5. Merge в main — по явной команде владельца.
