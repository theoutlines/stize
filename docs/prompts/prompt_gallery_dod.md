# Задача: галерея дизайн-слепков как часть Definition of Done

## Контекст

В репозитории есть `design/gallery/` — статичная HTML-галерея всех экранов и
состояний приложения (по странице на состояние), задеплоенная на Pages-алиас
`preview-gallery`. Она служит слепком текущего UI для работы над дизайном в
Figma. Чтобы галерея не протухала, её обновление должно стать частью
завершения каждой задачи, меняющей UI. Задача — встроить это правило в
процессные документы и оставить сессиям инструкцию по поддержанию галереи.

**Предусловие:** `feature/design-gallery` влита в main. Если нет —
**[STOP & ASK]**, задачу не начинать.

## Ветка

`fix/gallery-dod` от main, без отдельного worktree (правки только в доках).

## Что сделать

### 1. `CLAUDE.md` → раздел «Завершение задачи (definition of done сессии)»

Добавить пункт (между текущими 3 и 4, нумерацию поправить):

> Если задача добавляет или меняет видимые экраны/состояния UI — обновить
> соответствующие страницы в `design/gallery/` (добавить новые состояния,
> актуализировать изменённые, поправить `index.html`) и передеплоить
> `preview-gallery`. Правила — в `design/gallery/README.md`. Задачи без
> визуальных изменений галерею не трогают.

### 2. `WORKFLOW.md` → «Цикл задачи»

В схему цикла после «работа → отчёт…» вставить шаг
`→ обновить design/gallery при визуальных изменениях`. В раздел «Приёмка
задачи» добавить строку: сверять, что слепки галереи обновлены, если задача
меняла UI.

### 3. Создать `design/gallery/README.md`

Инструкция для будущих сессий (публичный текст):

- **Назначение:** галерея — слепок всех экранов и состояний UI для импорта в
  Figma (html.to.design разбирает DOM в слои); не продакшен-код, JS запрещён.
- **Когда обновлять:** новое видимое состояние → новая страница; изменение
  существующего экрана → правка его страницы; удалённая фича → страница
  удаляется из галереи и из `index.html`.
- **Правила страниц:** фрейм 390×844, kebab-case имена файлов, семантичные
  классы (имена слоёв в Figma), мок-данные Белграда, карта — статичная
  подложка + DOM-маркеры. Состояния из невлитых веток/спек помечаются в
  `index.html` бейджем `branch`/`spec`; после merge ветки бейдж снимается.
- **Деплой:** команда деплоя `preview-gallery` (взять фактическую из отчёта
  задачи design-gallery), выполняется сессией, не владельцем.
- **Отчётность:** в отчёте задачи, менявшей UI, — строка «Галерея: обновлены
  страницы …» либо «Галерея: визуальных изменений нет».

### 4. `docs/BACKLOG.md`

В раздел «В проде» добавить пункт про галерею дизайн-слепков (✅ после merge
design-gallery; статус сверить по факту).

## Отчёт

`docs/reports/YYYY-MM-DD-gallery-dod.md` (RU). Финальный раздел
**«Как проверить»**: где смотреть новые пункты в CLAUDE.md/WORKFLOW.md,
ссылка на `design/gallery/README.md`, URL галереи.

## Ограничения

- Только документы, код не трогать.
- Публичный текст, без секретов и инфра-деталей.
- Merge в main — по явной команде владельца.
