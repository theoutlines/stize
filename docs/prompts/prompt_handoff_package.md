# STIGLA — Хендофф-пакет для продуктового чата

> Выполнять ПОСЛЕ завершения задачи repo process setup. Задача на 5 минут: собрать
> в одну папку файлы, которые владелец отнесёт в продуктовый чат (Claude.ai Project).
> Ничего не менять по содержанию — только собрать и дописать один статусный файл.
> Отчёт — RU.

## Что сделать

1. Создать папку `handoff/` в корне репозитория и добавить её в .gitignore
   (это копии для выгрузки, не часть репо).

2. Скопировать в неё:
   - `CLAUDE.md` (уже с разделом «Процесс»);
   - `docs/BACKLOG.md`;
   - свежий отчёт repo setup из `docs/reports/` (самый последний по дате);
   - свежий отчёт аудита аналитики из `docs/reports/`, если он есть;
   - `STIGLA_VISION.md` — ТОЛЬКО если менялся за последние 3 месяца (проверить
     по `git log`); если не менялся — не копировать, а упомянуть это в STATUS.md.

3. Сгенерировать в `handoff/STATUS.md` (на русском) краткий срез текущего состояния:
   - список веток и worktrees: что в работе, что смержено, что пустое под будущее;
   - статус задачи аналитики: сделано / в работе / принята по чек-листу аудита;
   - нерешённые `[STOP & ASK]` из всех задач, если остались висеть;
   - незакрытые пункты безопасности (секреты на ротацию и т.п.), если были найдены;
   - последние 5–10 коммитов в main одной строкой каждый (что вообще происходило).

## Отчёт

Одна строка: путь к папке handoff/ и список файлов в ней.
