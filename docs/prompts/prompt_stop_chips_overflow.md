# Задача: сворачивание чипов линий на остановках с большим числом линий

Ветка: `fix/stop-chips-overflow` · Worktree: `../stigla-stop-chips-overflow`
Отчёт: `docs/reports/2026-07-XX-stop-chips-overflow.md` (RU)

## Контекст
После таргетного вливания пригорода (ветка gtfs-freshness) на краевых
пригородных узлах остановка несёт много линий — напр. **Baćevac ~22 линии**. Это
честное реальное пригородное сообщение (узел, где сходится много маршрутов), но
чипы-фильтр линий на экране остановки не рассчитаны на такое количество —
вероятно превращаются в кашу / клипаются / переполняют панель.

Маленькая изолированная UI-правка. НЕ трогает данные/бэкенд/рендер карты — только
как чипы линий показываются на экране остановки.

## Что сделать
1. Посмотреть текущие чипы-фильтр линий (`stop_screen.dart` / `stop_sheet.dart` —
   где строятся чипы линий остановки) на остановке с 15-22+ линиями. Оценить, что
   ломается: перенос, клип, переполнение, нечитаемость.
2. Сделать чипы устойчивыми к большому числу линий. Варианты (выбрать разумный):
   - горизонтальный скролл чипов (как уже сделано для favorites-carousel — F7,
     сверься с тем паттерном), либо
   - сворачивание с «+N ещё» / показать все по тапу, либо
   - многострочный wrap с ограничением высоты + скролл.
   Цель: на остановке с 22 линиями чипы читаемы, не клипаются, не ломают панель;
   на остановке с 2-3 линиями выглядят как раньше.
3. Проверить, что фильтрация по чипу (выбор линии) продолжает работать при любом
   числе линий.

## Границы
- Только UI чипов на экране остановки. НЕ трогать: данные, бэкенд, рендер карты,
  список прибытий (его логику), символьный слой, расписание.
- Не пересекаться с gtfs-freshness (данные/бэкенд) и symbol-layer (карта/движение)
  — это разные файлы; чипы — узкий UI-кусок.
- main релизопригоден; если правка визуально заметная — за флагом не обязательно
  (это исправление overflow, не фича), но не ломать существующий вид малых
  остановок.
- Код EN, отчёт RU.

## Definition of Done
1. Чипы читаемы и не клипаются на остановке с 22 линиями; малые остановки как
   раньше. Фильтрация работает при любом числе линий.
2. Тесты: widget-тест чипов при большом числе линий (не клипается, скролл/сворот
   работает).
3. Галерея: обновить состояние остановки с большим числом чипов (добавить
   страницу-пример «остановка с многими линиями») + передеплой preview-gallery.
4. Отчёт RU + «Как проверить» без терминала: staging-превью сессия деплоит сама;
   владельцу — открыть остановку Baćevac (или иную с 20+ линиями), убедиться,
   что чипы читаемы.
5. Merge в main — по явной команде владельца.
