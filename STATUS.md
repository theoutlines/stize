## Что сейчас в проде (2026-07-14)

- **Символьный слой ТС** (`symbol_layer`): движущиеся ТС рисуются как MapLibre
  GPU symbol layer (батчево, sub-linear по числу), вместо per-vehicle Flutter-
  виджетов. Виджет-путь остаётся фолбэком.
- **Движение ТС**: непрерывная экстраполяция вперёд между фиксами (метки не
  стоят 30 с), GPS-якорь к реальному фиксу (не «улетают»), сшивка по фактической
  геометрии направления. Таймд-траектория (`timed_trajectory`) — план движения с
  бэкенда, метки едут по времени.
- **Расписание-фолбэк**: список (`schedule_fallback`, Фаза 1) — экран остановки
  не пустует ночью/в межпик, плановые прибытия хвостом за живыми, помечены «По
  расписанию»; карта (`schedule_map`, Фаза 2) — где нет живого ТС, рисуются
  scheduled-объекты по timed-траектории.
- **Пригород**: пригородные линии влиты в фид (не только городские).
- **Остановки**: BUG-фикс рендера (кавычки в названиях — `jsonEncode` вместо
  geobase `toText()`, императивные слои); приоритет иконки типа (трамвайная
  остановка доминирует над авто/троллейбусом).
- **Кэш no-store** на нужных ответах; **direction-фикс** («ТС едет по домам» →
  `vehicle_direction_shape`, точный `route_id` направления с бэкенда).
- **Live-позиции** (`live_position_only`): на карте только ТС с реальным GPS;
  placeholder-строки (junk-гаражи `P1..P999`) остаются в списке, но не на карте.
- **Дата фида** в About (`/api/v1/gtfs-meta`).
- **«Рядом»** (`nearby_list` + `nearby_sort_board` = ON): draggable-лист линий
  вокруг геопозиции, живое + расписание (не пустует), группировка по
  `direction_route_id`, сортировка по времени-до-посадки. Расписание для
  ближайших N остановок (KV `config:nearby_schedule_stops`=5).

## Известные хвосты (не блокеры)
- **Edge «→ None» (41A)**: у части line-вариантов (`41A` и т.п.) `route_id` не
  резолвится в имя конечной в GTFS-метаданных → редкая строка «→ None» в «Рядом»
  (1/26 на плотнейшей точке). Нужен резолв имён line-вариантов.
- **404 shapes**: часть трамвайных линий (`_tramLines`, `map_support.dart:30`)
  запрашивает shape, которого нет → 404 в логах. Косметика/шум, не падение.
- **npm-аудит**: прогнать `npm audit` по backend-зависимостям, подчистить.
- **Fade застывших меток**: метки, потерявшие свежие фиксы, желательно плавно
  гасить, а не держать на месте.
- **KV free-tier лимит**: проект близок к лимитам free-tier (KV/CPU ~50 мс).
  Мониторить; при росте — план на платный тариф. «Рядом» специально капит
  расписание (K=5) ради CPU-запаса.

## Операционка (детали — в CLAUDE.md)
- Деплой backend прод: `cd backend && npm run deploy`. Staging: `npm run deploy:staging`.
- Деплой web прод: `flutter build web --release --dart-define-from-file=dart_defines.json`
  затем `wrangler pages deploy build/web --project-name=stigla --branch=main`.
- **Верификация web** — curl+sha256 `main.dart.js` кастомного домена vs локальный
  билд (браузер-кэш врёт; `pages.dev` и домен расходятся на пару минут).
- **Откат** — флагом в KV (`flag:<name>=0`), без деплоя.
- Staging-превью фронта: бейдж **STAGING** = бандл смотрит на staging-backend.
